## Design Context

### Users
Primary users are product, operations, and ML engineering teams managing intent pipelines and model behavior in a fast-moving internal environment. They use the interface during active operational work: monitoring health, updating intent libraries, and validating inference quality. The UI should reduce cognitive load while keeping dense technical information scannable.

### Brand Personality
Calm, precise, trustworthy. The interface should feel editorial and intentional rather than flashy, with visual confidence that signals reliability and operational maturity.

### Aesthetic Direction
Warm technical editorial: softly tinted neutrals, disciplined contrast, subtle depth, and restrained accent colors. Emphasis is on hierarchy, rhythm, and legibility. Anti-direction: generic dashboard templates, overuse of gradients/glows, excessive card nesting, and crowded all-primary controls.

### Design Principles
1. Build clear visual hierarchy through typography, spacing rhythm, and restrained emphasis.
2. Prefer calm, warm surfaces and subtle depth over loud effects.
3. Keep interaction language consistent: clear primary actions, secondary support actions, obvious states.
4. Optimize for scannability in data-heavy views (tables, metrics, form workflows).
5. Make polish systemic: shared tokens and reusable component styling, not one-off page hacks.
